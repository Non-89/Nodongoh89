# Blueprint: Lotto Lucky Number Generator

## **1. 개요 (Overview)**

- **목표:** 사용자에게 무작위 로또 번호를 생성하여 보여주는 세련되고 인터랙티브한 웹 애플리케이션입니다.
- **기술 스택:** 프레임워크 없는 순수 HTML, CSS, JavaScript를 사용하며, 최신 웹 표준(웹 컴포넌트, 모던 CSS/JS)을 준수합니다.

## **2. 현재까지 구현된 기능 (Features Implemented)**

### **버전 1.0**
- **기능:**
  - 'Generate Numbers' 버튼 클릭 시 6개의 로또 번호 1세트를 생성합니다.
- **디자인:**
  - 기본적인 HTML 구조와 스타일.

### **버전 2.0 (사용자 피드백 반영)**
- **기능:**
  - **[오해로 인한 잘못된 수정]** 6개가 아닌 5개의 번호를 생성하도록 변경했습니다.
- **디자인:**
  - 네온 효과가 적용된 제목, 그라데이션 버튼 등 고급스러운 다크 모드 디자인을 적용했습니다.
  - 번호(공)가 나타날 때 애니메이션 효과를 추가했습니다.

### **버전 3.0 (현재 목표)**
- **기능:**
  - 'Get Lucky Numbers' 버튼 클릭 시, **6개의 번호로 구성된 1줄짜리 로또 세트를 총 5줄 생성**합니다.
  - 각 줄과 각 번호 공이 순차적으로 나타나는 애니메이션을 적용합니다.
- **디자인:**
  - 5줄의 번호가 표시되어도 어색하지 않도록 전체 레이아웃을 수정합니다.
  - 컨테이너와 공의 크기를 조정하여 가독성을 높입니다.

## **3. 현재 작업 계획 (Current Plan)**

- **문제:** 이전 배포에서 코드 변경 사항이 실제 웹사이트에 반영되지 않는 지속적인 캐시(Cache) 문제가 발생하고 있습니다.
- **해결 전략:**
  1.  **파일 이름 변경 (Cache Busting):** 브라우저가 무조건 새로운 파일을 다운로드하도록 `style.css`와 `main.js` 파일의 이름을 각각 `style-v3.css`, `main-v3.js`로 변경합니다.
  2.  **`index.html` 수정:** 이름이 변경된 CSS, JS 파일을 `index.html`에 다시 연결합니다.
  3.  **최종 검수 및 배포:** 수정된 파일들을 최종적으로 확인하고 Firebase에 다시 배포합니다.
  4.  **사용자 안내:** 배포 완료 후, 사용자에게 변경 사항과 함께 확실한 확인을 위해 '강력 새로고침' 방법을 안내합니다.
